# Secrets Management

**Never commit secrets to this directory or anywhere in the repository.**

Secrets must be created out-of-band before deploying the manifests. Choose one of the following approaches:

---

## Option 1: kubectl (simple, manual rotation)

Create each required secret with the commands below. Replace placeholder values with real secrets generated via `openssl rand -hex 32`.

```bash
# JWT signing secret (minimum 32 chars, prefer 64)
kubectl create secret generic jwt-secret \
  --namespace medical-ocr \
  --from-literal=jwt-secret="$(openssl rand -hex 32)"

# PHI master encryption key (must be exactly 64 hex characters = 32 bytes AES-256)
kubectl create secret generic phi-master-key \
  --namespace medical-ocr \
  --from-literal=phi-master-key="$(openssl rand -hex 32)"

# Database credentials
# Replace <DB_PASSWORD> with the same password used when creating the postgres pod.
kubectl create secret generic db-credentials \
  --namespace medical-ocr \
  --from-literal=db-password="<DB_PASSWORD>" \
  --from-literal=db-url="postgresql://medical:<DB_PASSWORD>@postgres:5432/medical_ocr"

# Initial admin password (rotated after first login)
kubectl create secret generic initial-admin \
  --namespace medical-ocr \
  --from-literal=initial-admin-password="<INITIAL_ADMIN_PASSWORD>"

# Mosquitto TLS certificates
# Copy the certs generated by scripts/gen-dev-certs.sh (or your PKI)
kubectl create secret generic mosquitto-tls \
  --namespace medical-ocr \
  --from-file=ca.crt=infrastructure/mosquitto/certs/ca.crt \
  --from-file=api_server.crt=infrastructure/mosquitto/certs/api_server.crt \
  --from-file=api_server.key=infrastructure/mosquitto/certs/api_server.key

# TLS certificate for the Ingress
kubectl create secret tls medical-ocr-tls \
  --namespace medical-ocr \
  --cert=path/to/tls.crt \
  --key=path/to/tls.key
```

## Option 2: Sealed Secrets (GitOps-friendly)

Install the [Sealed Secrets](https://github.com/bitnami-labs/sealed-secrets) controller, then encrypt secrets with `kubeseal`:

```bash
kubectl create secret generic jwt-secret \
  --namespace medical-ocr \
  --from-literal=jwt-secret="$(openssl rand -hex 32)" \
  --dry-run=client -o yaml \
  | kubeseal --format yaml > infrastructure/kubernetes/secrets/jwt-secret-sealed.yaml
```

The `*-sealed.yaml` files are safe to commit. The controller decrypts them at deploy time.

## Option 3: External Secrets Operator (recommended for cloud deployments)

Use [External Secrets Operator](https://external-secrets.io/) with a supported backend:
- **AWS Secrets Manager** — `SecretStore` with `provider: aws`
- **HashiCorp Vault** — `SecretStore` with `provider: vault`
- **GCP Secret Manager** — `SecretStore` with `provider: gcpsm`
- **Azure Key Vault** — `SecretStore` with `provider: azurekv`

Example `ExternalSecret` for JWT secret stored in AWS Secrets Manager:

```yaml
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: jwt-secret
  namespace: medical-ocr
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: aws-secrets-manager
    kind: SecretStore
  target:
    name: jwt-secret
    creationPolicy: Owner
  data:
    - secretKey: jwt-secret
      remoteRef:
        key: medical-ocr/jwt-secret
        property: value
```

## Required secrets summary

| Secret name       | Keys                          | Used by        |
|-------------------|-------------------------------|----------------|
| `jwt-secret`      | `jwt-secret`                  | API            |
| `phi-master-key`  | `phi-master-key`              | API            |
| `db-credentials`  | `db-password`, `db-url`       | API, Postgres  |
| `initial-admin`   | `initial-admin-password`      | API            |
| `mosquitto-tls`   | `ca.crt`, `api_server.crt`, `api_server.key` | API, Mosquitto |
